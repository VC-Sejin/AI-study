# 요구사항정의서 작성 프롬프트 (재사용 템플릿)

새 회사의 RFP로 요구사항정의서 엑셀을 만들 때, 아래 전체를 복사해 붙여넣고 RFP 파일을 첨부한 뒤 "이 양식대로 만들어줘"라고 요청하세요.

사용법 요약: RFP만 바꿔 끼우면 같은 표 구조가 나옵니다. 양식(칼럼 구조)과 데이터(RFP 내용)를 분리해두었기 때문입니다.

---

## 프롬프트 본문 (이 아래를 복사)

```
[역할]
너는 SI/UX 프로젝트의 요구사항정의서 작성 담당이야. 첨부한 RFP(제안요청서)를 읽고,
아래 양식 규칙에 정확히 맞춰 요구사항정의서 엑셀(.xlsx)을 만들어줘.

[입력]
- 첨부: 회사명_RFP.pdf (또는 hwp/워드 텍스트)

[표 구조 — 2단 헤더, 왼쪽부터 순서대로]
1) No : 1부터 세로 일련번호
2) 대제목 "요청 정보"(셀 병합) 하위:
   요구사항 ID, RFP ID, 요청단계, 요청경로, 요청일자, 요청자
3) 대제목 "요구사항"(셀 병합) 하위:
   유형, 대분류, 중분류, 소분류, 담당자, 요구사항명, RFP요건, 추가내용, 신규/개선, 적용 범위
4) 대제목 "협의내용 (1차)"(셀 병합) 하위: 구분, 제약사항, 해결방안 및 대안
5) 대제목 "협의내용 (2차)"(셀 병합) 하위: 구분, 제약사항, 해결방안 및 대안
6) 대제목 "고객사 확인"(셀 병합) 하위: 수용여부, 확인일자, IT검토자, 현업검토자

[채우는 칸 / 비우는 칸]
- 채움: No, 요구사항 ID, RFP ID, 유형, 대분류, 중분류, 요구사항명, RFP요건, 신규/개선, 적용 범위
- 빈칸(작성용): 요청단계·요청경로·요청일자·요청자, 소분류, 담당자, 추가내용,
  협의내용(1·2차) 전체, 고객사 확인 전체

[ID 규칙]
- RFP ID : RFP 원문의 고유번호 그대로 (예: SFR-001, PER-003). 추적용이니 절대 변형 금지.
- 요구사항 ID : 우리 내부 ID. 기능=FR-001.., 비기능=NFR-x.., 사업관리=계열별(SYS/MGT/ORG/EDU/PAY)

[유형(구분) 분류 규칙]
- 기능 : 사용자/관리자가 쓰는 화면·기능 (RFP의 SFR 등). IA·화면설계 직접 재료.
- 비기능 : 성능·테스트·보안·품질·데이터 등 전체 공통 기준 (PER/TER/SER/QUR/DAR 등)
- 사업관리 : 시스템구성·사업추진·수행조직·교육·대금 등 계약/관리 성격 (ECR/BPR/POR/EDR/PAT 등)
  ※ RFP마다 ID 규칙이 다르니, 먼저 RFP의 "요구사항 개요/구분표"를 찾아 그 분류를 따를 것.

[추론 규칙]
- 적용 범위 : 상세설명 근거로 앱/웹/어드민/서버 등 추론. 단정 어려우면 "공통".
- 신규/개선 : RFP에 (신규)/(개선) 표기가 있으면 그대로, 없으면 "-".
- 임의 추론값(적용범위 등)은 마지막에 "가정·확인 필요" 목록으로 따로 알려줄 것.

[누락 금지]
- RFP 요구사항 개요표의 총 건수와 내가 만든 행 수가 반드시 일치해야 함.
  일치하지 않으면 어디가 빠졌는지 보고할 것.

[서식]
- 폰트 맑은 고딕, 대제목 병합, 유형별 색상(기능/비기능/사업관리), 자동필터, 틀 고정.
- 수식 오류 0, 완성 후 .xlsx로 전달.
```

## 주의할 점

- RFP마다 ID 규칙과 구분 체계가 다릅니다. 이번 현대백화점 RFP는 ECR/SFR/PER 식이었지만, 다른 회사는 REQ-001이나 F-01처럼 쓰기도 합니다. 그래서 프롬프트에 "먼저 RFP의 요구사항 개요표를 찾아 그 분류를 따르라"는 규칙을 넣어 두었습니다.
- 우선순위나 관련 사용자처럼 이 양식에 칸이 없는 정보가 필요하면, '추가내용' 칸에 적도록 프롬프트에 한 줄 추가하면 됩니다.
- 협의내용을 1차/2차로 나누지 않고 하나로 쓰려면, 프롬프트의 4)·5) 항목을 "협의내용" 한 그룹으로 합치면 됩니다.
