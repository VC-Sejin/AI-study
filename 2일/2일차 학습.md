# 2일차 학습 — WBS 생성 개요

## 1. WBS란?

**WBS(Work Breakdown Structure, 작업 분해 구조)** 는 프로젝트의 최종 산출물을 달성하기 위해 수행해야 할 모든 작업을 계층적으로 분해한 구조도이다. "무엇을 해야 하는가"를 관리 가능한 단위로 나누어, 일정·비용·자원 관리의 기준선(Baseline)을 제공한다.

> 핵심 원칙: **"작업(Work)을 분해하는 것이지, 사람이나 시간을 분해하는 것이 아니다."**

---

## 2. 생성 목적

| 목적 | 설명 |
|------|------|
| 범위 명확화 | 프로젝트에 포함되는 작업과 제외되는 작업을 명확히 구분 |
| 일정·비용 산정 | 작업 단위별 공수 추정 및 일정 수립의 기초 |
| 책임 분배 | 작업별 담당자(R&R) 지정 근거 마련 |
| 진척 관리 | 완료 기준이 명확한 단위로 진행률 측정 |

---

## 3. 구성 요소 (계층 구조)

```
프로젝트 (Level 0)
└─ 주요 산출물/단계 (Level 1)
   └─ 하위 산출물 (Level 2)
      └─ 작업 패키지 (Work Package, 최하위 관리 단위)
```

- **작업 패키지(Work Package)**: WBS의 최하위 단위. 일정·비용을 추정하고 담당자를 지정할 수 있는 가장 작은 관리 단위.

---

## 4. 생성 절차

1. **최종 산출물 정의** — 프로젝트가 만들어 낼 결과물을 명확히 한다.
2. **주요 단계/산출물 분해** — 최종 산출물을 큰 덩어리(Level 1)로 나눈다.
3. **하위 작업으로 세분화** — 각 단계를 더 작은 작업으로 반복 분해한다.
4. **작업 패키지 도출** — 추정·할당이 가능한 수준까지 분해한다.
5. **코드 부여 및 검증** — WBS 코드(예: 1.2.3)를 부여하고 누락·중복을 점검한다.

---

## 5. 작성 원칙

- **100% 규칙**: 상위 항목은 하위 항목의 합으로 100% 구성되어야 한다 (누락·초과 금지).
- **상호 배타성(MECE)**: 작업 간 중복이 없어야 한다.
- **산출물 중심**: 활동이 아닌 "결과물" 기준으로 분해한다.
- **적정 분해 수준**: 너무 잘게(관리 부담) 또는 너무 크게(추정 부정확) 나누지 않는다 (보통 8/80 시간 기준).

---

## 6. 표현 방식

| 형태 | 특징 |
|------|------|
| 트리형(조직도형) | 계층 관계를 시각적으로 파악하기 쉬움 |
| 목록형(들여쓰기) | 항목이 많을 때 관리·편집이 용이 |
| 표/스프레드시트형 | 일정·담당자·공수 등 속성 관리에 적합 |
