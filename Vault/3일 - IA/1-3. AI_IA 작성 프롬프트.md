최종 IA를 한 번에 재현하기 위한 프롬프트를 정리해 드리겠습니다. 지금까지 6차에 걸쳐 누적된 요구사항(개발 제외 → 표준 커머스 구조 → 하위 세분화 → 계단식 양식 → 비고 분리·출처 표기)을 하나의 완결된 지시문으로 합치는 게 핵심입니다.

채팅에서 바로 복사해 쓰실 수 있도록 프롬프트 본문을 제시합니다.

### 최종 IA 산출용 프롬프트

```
[역할]
너는 커머스 플랫폼 IA(정보구조도) 설계자다. 첨부한 두 자료를 분석해
표준 커머스 앱 구조 기반의 IA 정의서를 Excel(.xlsx)로 산출한다.

[입력 자료]
1) 요구사항정의서 (FO UI/UX) — 최신 버전 시트만 사용 (예: "요구사항 정의서 (V0.8)")
2) 제안요청서(RFP)

[전처리]
- 요구사항정의서에서 NO·요구사항ID가 그룹 첫 행에만 있는 병합 셀은 아래로 채워(forward-fill) 처리.
- RFP 본문을 파싱해 "고유번호(SFR-xxx 등) → 요건 상세설명" 매핑 사전을 만든다.
- 유형이 '개발'인 행(검색엔진 색인, DB 작업, 백엔드 연동 등 비화면 항목)은 전부 제외한다.

[IA 구조 — 표준 커머스 앱 골격을 계단식(cascading)으로 구성]
- 1depth = 영역: FO(고객 서비스) / BO(운영 어드민)
  · 어드민/관리자/등록·수정·승인/HM·HBO 코드 또는 유형='어드민' → BO, 그 외 → FO
- 2depth = 표준 메뉴
  · FO: 홈/메인, 카테고리/전시, 검색, 상품상세, 장바구니, 주문/결제, 마이페이지,
        회원/인증, 혜택(이벤트/프로모션/쿠폰), 연동/대응, 공통
  · BO: 전시/매장 관리, 검색 관리, 상품 관리, 주문/배송/CS 관리, 회원 관리,
        이벤트 관리, 프로모션/쿠폰/캠페인 관리, 운영 공통
- 3depth = 세부 메뉴 (키워드 기반 분리). 예:
  · 주문/결제 → 주문서 / 결제·PG / 취소·교환·반품 / 배송·물류 / O4O·매장 주문 / 무형상품 주문 / 선물하기
  · 카테고리/전시 → 브랜드관 / 전문관·테마관 / 카테고리 매장 / 기획전 / 광고·배너 구좌 / 페이지 빌더 / 콘텐츠·라이브 / 특화관
  · 검색 → 검색 영역 / 검색 결과 / 필터·정렬 / 검색어·추천어 / 검색 수단
  · 회원/인증 → 로그인·인증 / 회원가입·통합 / 등급 / 포인트·적립금 / 멤버십 / 정보·설정
- 4~6depth = 정의서의 소분류·요구사항명 (상위 depth에 포함되는 중복 마디는 생략)
- 계단식 표기: 상위 메뉴는 그룹 첫 행에만 쓰고, 하위는 그 아래로 내려가며 빈칸으로 들여쓴다.
  ※ 복합 모듈(예 "주문/상품")은 첫 토큰 기준으로 분류한다.

[컬럼 구성]
NO | 구분 | 1depth | 2depth | 3depth | 4depth | 5depth | 6depth |
화면 ID | 요구사항 ID | 요구사항 ID 근거 | 화면 설명 | 생성 구분

- 구분: 화면 / 팝업 / 프로세스
  · 팝업·모달·알럿·레이어 → 팝업, API·연동·배치·스케줄 → 프로세스, 나머지 → 화면
- 화면 ID: 신규 생성. 형식 = 영역코드(F/B) + 메뉴코드 + 일련번호 (예: F-ORDR-001)
- 요구사항 ID: 정의서의 RFP 고유번호. 없으면 "ID 없음"
- 요구사항 ID 근거: 해당 고유번호의 RFP 본문 요건 상세를 그대로 인용.
  본문이 없으면 "(RFP 본문에 상세 없음)", 고유번호 자체가 없으면 정의서 메모 기재.
- 화면 설명: 정의서 세부내용 인용.
- 생성 구분(출처 표기):
  · 화면ID·구분·1~3depth = IA 임의생성
  · 요구사항ID·근거·화면설명·4~6depth = 원본(RFP/정의서) 인용
  · 행별로 "원본(RFP/정의서) + IA분류 임의생성" 또는
    "IA분류 임의생성 / 요구사항ID 미지정(정의서 메모)" 명시

[서식]
- 참고 양식: 계단식 depth + depth별 음영으로 위계 가독성 확보.
- 임의생성 값은 빨간 글씨, 원본 인용은 회색, 생성구분은 녹색.
- 요구사항ID 미지정 행은 노란 음영으로 강조.
- 상단에 출처 표기 범례를 넣는다.
- 맑은 고딕, 머리행 고정, 자동필터, 수식 오류 0건.

[출력]
- 단일 시트 "IA정의서"로 .xlsx 산출.
- 작업 완료 후 영역별 행 수(FO/BO)와 표준 메뉴별 분포를 요약 보고.
```

만약, 다른 도메인이라면...?

### 재사용 가능한 범용 프롬프트
A를 영화 예매 앱으로 변환하여 작성해주고, 결과 기반으로 IA작성해줘

```
[역할]
너는 커머스/서비스 IA(정보구조도) 설계자다. 첨부 자료를 분석해
표준 IA 정의서를 Excel(.xlsx)로 산출한다.

────────────────────────────────────
[A. 도메인 파라미터]  ← 프로젝트마다 이 블록만 교체
- 도메인/서비스명: {{예: 커머스 앱, OTT, 예약 플랫폼, 사내 어드민}}
- 영역 구분: {{예: FO(고객)/BO(운영), 또는 사용자/관리자/파트너}}
- 1depth 표준 메뉴(영역별로 나열):
    · FO: {{홈, 탐색/카테고리, 검색, 상세, 장바구니, 주문/결제, 마이페이지, 회원, 혜택 …}}
    · BO: {{전시 관리, 상품 관리, 주문 관리, 회원 관리, 프로모션 관리 …}}
- 3depth 세부 메뉴 분리 키워드: {{메뉴별로 묶을 키워드 군}}
- 제외할 항목 기준: {{예: 유형='개발', 비화면 백엔드/연동/색인 항목}}
- 요구사항 ID 체계: {{예: SFR-xxx, REQ-xxx — RFP/기획서의 고유번호 규칙}}
- 화면 ID 코드 규칙: {{영역코드 + 메뉴코드 + 일련번호}}
────────────────────────────────────

[B. 방법론]  ← 도메인 무관, 항상 동일
1) 전처리
   - 정의서에서 그룹 첫 행에만 있는 병합 셀(NO·요구사항ID 등)은 forward-fill.
   - 요건 원문을 파싱해 "고유번호 → 요건 상세설명" 매핑 사전 생성.
   - [A.제외 기준]에 해당하는 행은 모두 제외.
2) 분류 (계단식 cascading 구조)
   - 1depth=영역, 2depth=표준 메뉴, 3depth=세부 메뉴, 4depth~=정의서 소분류·요구사항명.
   - 분류는 [A]의 메뉴·키워드 목록을 우선 적용하고,
     목록에 없으면 의미상 가장 가까운 표준 메뉴로 배정(없으면 '기타/공통').
   - 상위 depth에 포함되는 중복 마디는 하위에서 생략.
   - 복합 모듈(예 "A/B")은 첫 토큰 기준 분류.
   - 상위 메뉴는 그룹 첫 행에만 표기, 하위는 빈칸 들여쓰기.
3) 컬럼
   NO | 구분 | 1depth … Ndepth | 화면 ID | 요구사항 ID | 요구사항 ID 근거 | 화면 설명 | 생성 구분
   - 구분: 화면 / 팝업 / 프로세스 (팝업·모달·알럿→팝업, API·연동·배치→프로세스, 그 외→화면)
   - 화면 ID: [A.코드 규칙]대로 신규 생성.
   - 요구사항 ID: 원문 고유번호. 없으면 "ID 없음".
   - 근거: 해당 고유번호의 원문 요건 인용. 없으면 "(원문에 상세 없음)" 또는 정의서 메모.
   - 설명: 정의서 세부내용 인용.
4) 출처 표기 (생성 구분 컬럼) — 도메인 무관, 항상 적용
   - 임의생성: 화면ID·구분·1~3depth(영역/표준메뉴/세부메뉴)
   - 원본 인용: 요구사항ID·근거·화면설명·4depth 이하
   - 행별로 "원본 + IA분류 임의생성" / "IA분류 임의생성·요구사항ID 미지정" 명시.

[C. 서식]
- 계단식 depth + depth별 음영, 임의생성=빨강·원본=회색·생성구분=녹색, 미지정 행=노란 음영.
- 상단 출처 범례, 머리행 고정, 자동필터, 수식 오류 0건, 단일 시트.

[D. 출력]
- .xlsx 산출 후 영역별 행 수와 표준 메뉴별 분포를 요약 보고.
- [A]의 메뉴 목록으로 분류되지 않아 '기타'로 빠진 항목은 따로 리스트업해 보고.
```

### 도메인 전환 시 작성 요령

**바꾸는 건 [A] 블록뿐입니다.** 예를 들어:

- **OTT 서비스** → 영역: 시청자/운영자, 1depth: 홈·카테고리·검색·콘텐츠 상세·내 보관함·구독/결제·계정 / 콘텐츠 관리·편성 관리·회원 관리
- **음식 배달** → 1depth: 홈·카테고리·검색·매장 상세·장바구니·주문/결제·주문내역·마이페이지 / 가맹점 관리·주문 관리·정산 관리
- **사내 어드민** → 영역 구분 자체를 권한별(일반/관리자/시스템)로 재정의
