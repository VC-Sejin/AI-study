## WBS 수립 — 작업 로그

문서 출처: 더현대 Hi / 영화예매앱 IA 정의서, 바이널씨 OSSP·표준 WBS 양식 정리일: 2026-06-12

### 1. 사업 개요

- 사업명: 더현대 Hi 커머스 플랫폼 / 영화 예매 앱 WBS 수립
- 입력 자료: 각 프로젝트 IA 정의서, 바이널씨 OSSP 프로세스 정의서(V4.1), 표준 WBS 양식(.xlsm 3종·SK브로드밴드 PDF)
- 목표: IA 기반의 단순 간트에서 출발해, 바이널씨 표준 양식을 적용한 진척관리형·수식자동화 WBS로 완성

핵심 방향은 IA 화면 구조를 일정으로 환산, 바이널씨 표준 양식 준수(일/영업일·WBS 4단 계층), 착수일·가중치·진척율 수식 자동화, 변경관리대장 기반 운영 체계 마련입니다.

### 2. 사업 대상 및 범위

대상 프로젝트는 두 건입니다. 더현대 Hi는 FO(고객 서비스) 297화면 / BO(운영 어드민) 102화면, 총 399화면 규모이고, 영화 예매 앱은 FO(고객 앱) 101화면 / BO 19화면, 총 120화면 규모입니다.

작업 범위는 WBS 수립과 양식 표준화입니다. 일정 골격 설계, OSSP 프로세스 반영, 바이널씨 양식 적용, 수식 자동화(날짜·업무구성비·진척율), 변경관리대장 추가가 포함됩니다. 실제 개발·디자인 실행은 범위 밖입니다.

### 3. 입력 자료 구성

작업에 사용한 입력 자료는 다음과 같습니다.

|구분|자료|용도|
|---|---|---|
|IA|IA_영화예매앱_v0_2|영화예매앱 화면·메뉴 도출|
|IA|더현대Hi IA_v0_5|더현대 화면·메뉴 도출|
|프로세스|바이널씨 OSSP 프로세스 정의서 V4.1 (Task 64)|단계·작업명 표준|
|양식|SKT MVNO·NewPoC WBS, 템플릿 .xlsm|WBS 표준 양식(계층·진척·간트)|
|양식|SK브로드밴드 WBS PDF|일정 표현 참고|

### 4. WBS 단계 분류 — 일정의 직접 재료

IA에서 도출한 화면군을 일정 단위로 묶어 10개 대분류로 구성했습니다. 준비, 분석, 화면기획, 디자인, 공통/기반 개발, FO 개발, BO 개발, 외부연동·데이터이관, 통합/테스트, 오픈/이행입니다. 각 대분류 아래 중분류·작업·세부작업(메뉴 단위)으로 4단 전개했고, 세부작업은 IA의 표준메뉴별 화면 수를 그대로 표기했습니다(예: 더현대 주문/결제 86화면, 영화예매앱 예매 플로우 24화면).

### 5. 양식 표준 — 전 항목 공통 기준

바이널씨 표준 양식의 공통 사양을 두 WBS에 동일하게 적용했습니다. 일/영업일(WORKDAY) 기준으로 주말을 자동 제외하고, WBS코드와 개요수준(1 대분류·2 중분류·3 작업·4 세부작업) 계층을 따르며, 착수일 셀 하나로 전체 날짜가 재계산됩니다. 상대비중을 입력하면 업무구성비가 자동 정규화(합 100% 유지)되고, 진척율 입력 시 대분류·전체 가중 진척율이 SUMPRODUCT로 집계됩니다. 간트는 조건부 서식으로 자동 표시됩니다.

### 6. 관리 성격 항목 — 별도 구성

변경관리(CCB), 작성지침은 일정 산출과 무관한 관리 성격 항목으로 별도 시트로 분리했습니다. 변경관리대장은 변경 유형·내용·사유·영향 WBS·일정/가중치 영향·승인여부를 기록하며, 드롭다운으로 입력합니다. 작성지침은 자동 계산 방식과 변경 반영 절차를 안내합니다.

### 7. 진행한 작업

1. 영화예매앱 IA(v0.2)로 8단계 주간 간트 WBS 초안을 작성했습니다.
2. 더현대 Hi IA(v0.5)를 분석(399화면)해 동일 방식의 간트를 만들고, 기간을 2025.02~2026.03(14개월)으로 조정했습니다.
3. 바이널씨 OSSP 프로세스 정의서(PDF, Task 64)를 편집용 엑셀로 변환했습니다.
4. OSSP 5단계·작업명을 차용해 더현대 WBS를 재구성한 뒤, 번호 체계를 빼고 1~8단계로 정리했습니다.
5. 일정 가정(분석/기획 5개월, 디자인 7개월 동시 시작, 와이어프레임 삭제)을 반영했습니다.
6. 바이널씨 표준 양식(.xlsm·PDF)을 분석하고, 더현대 WBS를 일/영업일·4단 계층·진척관리형 V1.0으로 재작성했습니다.
7. 착수일 기준 WORKDAY 날짜와 가중 진척율 SUMPRODUCT 수식을 적용했습니다.
8. 변경관리대장(CCB)과 작성지침 시트를 추가했습니다.
9. 상대비중 입력 기반 업무구성비 자동 정규화(합 1.0 유지)를 적용했습니다.
10. 간트를 조건부 서식으로 전환해 행 추가·삭제 시 자동 반영되게 했습니다.
11. 동일 양식을 영화예매앱 IA(120화면)에 적용해 V1.0을 만들었습니다.

### 8. 판단·가정 사항 (확정 필요)

- 화면 수는 각 IA 정의서의 집계 실값이지만, 일정 배분(착수 시점·기간)과 업무구성비는 표준 가정에 따른 추정치입니다. 실제 투입 인력과 착수일에 맞춰 확정이 필요합니다.
- 착수일은 2025-02-03(월)을 기준점으로 가정했습니다. 실제 계약 시점에 맞춰 셀에서 변경하면 전체 일정이 재계산됩니다.
- 일정 가정(분석 5개월·디자인 7개월 등)은 협의 과정에서 제시된 값으로, 변경 시 후속 단계가 연동되어 밀립니다.
- 더현대의 공통/기반(페이지빌더·회원통합), 영화예매앱의 예매 트랜잭션 엔진을 선행 작업으로 배치한 것은 표준적 의존관계 가정에 따른 것입니다.

### 9. 다음 단계 (예정)

요구사항정의서가 갱신되면 변경관리대장에 먼저 등록한 뒤, 영향 WBS 행의 착수·기간·상대비중을 조정하고 버전을 올립니다(V1.0 → V1.1). 추가로 검토할 사항은 선후행 의존관계(마일스톤) 명시 여부, 원본 .xlsm 매크로 틀에 데이터를 이식할지 여부, 그리고 두 WBS의 담당·투입 인력(M/M) 산정입니다.
