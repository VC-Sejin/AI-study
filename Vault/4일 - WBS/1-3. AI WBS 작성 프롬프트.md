**[복사용 프롬프트 — 진척관리 대시보드 포함 WBS]**

> 첨부한 IA 정의서로 WBS를 엑셀로 만들어줘. 바이널씨 표준 WBS 양식을 따르고 아래 사양을 정확히 지켜줘.
> 
> **시트 구성**: ① 일정 ② 진척관리(대시보드) ③ 변경관리대장 ④ 작성지침.
> 
> **일정 시트 — 컬럼**: WBS코드 / 개요수준(1 대분류·2 중분류·3 작업·4 세부작업) / 작업 / 착수(영업일) / 기간(일) / 시작일 / 종료일 / 상대비중 / 업무구성비 / 진척율 / 예상진척 / 진척차이 / 상태 / 담당. 그 오른쪽에 주 단위 간트.
> 
> **계층**: 10개 대분류(준비·분석·화면기획·디자인·공통/기반 개발·FO개발·BO개발·외부연동·통합/테스트·오픈)로 묶고, 설계·개발 단계는 IA의 표준메뉴별로 세부작업(4수준)을 한 줄씩 펼쳐 각 줄에 화면 수를 괄호로 표기.
> 
> **수식 자동화** (전부 하드코딩 말고 수식):
> 
> - 상단에 착수일·기준일 입력 셀. 시작일 = WORKDAY(착수일-1, 오프셋+1), 종료일 = WORKDAY(시작일-1, 기간)으로 영업일 계산.
> - 업무구성비 = 상대비중 ÷ 전체 상대비중 합(자동 정규화, 합 100%). 분모는 넓은 고정 범위로 잡아 행 추가·삭제에도 유지.
> - 진척율 입력 시 대분류·전체 가중 진척율을 SUMPRODUCT로 집계.
> 
> **진척관리(계획 대비 실적)**:
> 
> - 예상진척 = 기준일이 작업 시작 전 0%, 종료 후 100%, 사이는 경과 영업일 비율.
> - 진척차이 = 실적 − 예상(음수 빨강, 0 이상 초록).
> - 상태 = 완료/진행중/미착수 자동 판정, 종료일 지났는데 미완이면 '지연'(빨강 배경). 완료 초록, 진행중 노랑.
> - 상단에 전체 실적/계획 진척율을 나란히.
> 
> **진척관리 시트(대시보드)**: 일정 시트의 대분류 행과 기준일을 **수식으로 자동 참조**해서, 10개 단계별로 업무구성비·계획 진척·실적 진척·진척차이·달성률(실적÷계획)·상태를 요약. 맨 위에 기준일·전체 계획·전체 실적 표시, 맨 아래 합계행. 진척차이·상태에 위와 같은 조건부 서식. 데이터 입력은 일정 시트에서만 하고 대시보드는 집계만 하도록.
> 
> **간트**: 셀 색칠이 아니라 조건부 서식으로, 각 주 컬럼이 그 행 시작일~종료일 사이면 단계별 색 자동 표시(행 추가·삭제 자동 반영).
> 
> **표시·기타**: 사람이 바꾸는 셀(착수일·기준일·상대비중·진척율)은 파란 글씨, 수식은 검은 글씨. 폰트 맑은 고딕, 단계별 색 구분, 틀 고정. 착수일은 (날짜)로 가정. 변경관리대장은 변경유형·우선순위·승인여부 드롭다운. 만든 뒤 수식 오류 0개인지 검증하고, 파일명은 영문으로.
> 
> 화면 수는 IA에서 실제 집계하고, 일정·가중치는 표준 가정 추정치로 채운 뒤 그 사실을 알려줘.
